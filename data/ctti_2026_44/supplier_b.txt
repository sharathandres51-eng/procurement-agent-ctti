=====================================================================
EXECUTIVE SUMMARY
=====================================================================

NexCloud Iberia SA submits this technical proposal for CTTI tender CTTI-2026-44.
Our firm holds ENS ALT certification — the highest tier — for our Spanish public
cloud service, making us one of a small number of providers in the market able
to offer certified infrastructure for the most sensitive Generalitat workloads.
Our proposal prioritises security architecture and regulatory compliance, areas
in which we believe our credentials are unmatched in this procurement.

=====================================================================
ARQUITECTURA TÈCNICA PROPOSADA
=====================================================================

The proposed architecture uses Microsoft Azure as the primary hyperscaler, with
a BSC private cloud segment for ENS ALT workloads. Connectivity between Azure
and BSC is provided via ExpressRoute with private peering.

CRITICAL tier services are deployed with active-passive failover across two
Azure availability zones. The proposed RTO for CRITICAL services is 4 hours,
meeting the minimum PPT requirement. RPO is proposed at 1 hour, meeting the
minimum requirement. Failover is described as automatic in the proposal narrative,
though the specific Azure Traffic Manager configuration and health check thresholds
are not detailed in this document.

Identity federation is implemented via Azure AD Connect with SAML 2.0 support.
The proposal does not address in detail how existing CTTI Active Directory group
policy objects will be migrated or replicated to the cloud environment, nor does
it specify the API compatibility strategy for services with existing consumers
during the cutover period.

Monitoring integration is proposed via Azure Monitor with export to CTTI
collection platforms. The specific protocols and format of metric and log
forwarding are not specified in this proposal.

=====================================================================
SEGURETAT I COMPLIMENT NORMATIU
=====================================================================

NexCloud Iberia holds ENS ALT certification for its Spanish cloud service
(certification reference NX-ENS-ALT-2025-014, valid until March 2027). This is
the highest ENS certification tier and covers all 75 MEDI controls plus the
additional 48 ALT-specific controls. Certification documentation is included as
Annex A. This distinguishes our offer from competitors who hold only MEDI
certification: CTTI's highest-sensitivity workloads can be placed on NexCloud
infrastructure without requiring migration to the BSC private cloud.

GDPR compliance architecture uses customer-managed encryption keys via Azure
Key Vault under exclusive CTTI control. All personal data in scope is stored
within the EEA. Data flow documentation will be delivered within 60 days of
contract start. A DPA compliant with GDPR Article 28 is included in the
commercial annex.

Data sovereignty measures include Azure Policy assignments preventing replication
to non-EEA regions, with continuous compliance monitoring via Microsoft Defender
for Cloud. For workloads at BSC, physical residency within the BSC Barcelona
facility is guaranteed by the hosting agreement.

Security monitoring integration includes export of security events to the Agència
de Ciberseguretat de Catalunya's SIEM platform via CEF format over syslog-TLS,
from day one of each migration wave.

=====================================================================
PLA DE TRANSICIÓ DEL SERVEI
=====================================================================

The service handover plan commits to two structured knowledge transfer sessions,
meeting the minimum PPT requirement of two sessions of 8 hours each. Sessions
will cover system architecture and operational procedures respectively. All
documentation will be delivered in Catalan.

A 6-month post-handover support period is included in the base proposal. Extension
to 12 months is available as a priced option in the Sobre C submission. The
transition team composition — specifically the number of engineers dedicated
exclusively to handover activities — is not specified in this document; the
proposal states that resources will be assigned from the project team at the
time of handover.
